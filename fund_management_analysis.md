# 🏠 মেস ফান্ড ম্যানেজমেন্ট — বর্তমান সিস্টেম ও সমাধান প্রস্তাবনা

---

## 📊 বর্তমান সিস্টেম কিভাবে কাজ করছে

### ১. টাকা জমা (Deposits)
- প্রতিটি সদস্য ম্যানেজারের কাছে টাকা জমা দেয়
- জমার টাকা `deposits` কালেকশনে Firebase-এ সংরক্ষিত হয়
- সব সদস্যের জমার মোট = **ফান্ডে মোট জমা** (`totalDeposit`)

### ২. খরচের ধরন ৩ প্রকার

| ধরন | কোডে টাইপ | ব্যাখ্যা | ফান্ড থেকে কাটে? |
|------|-----------|----------|------------------|
| **সাধারণ বাজার** | `regular` | সদস্য বাজার করে, ফান্ড বা নিজের পকেট থেকে | `fundSource` অনুযায়ী |
| **দোকানে বকেয়া** | `baki` | দোকানে বাকি রাখা হয়েছে, পরে দিতে হবে | ❌ না (বকেয়া হিসেবে ট্র্যাক হয়) |
| **অতিরিক্ত খরচ** | `additional` | সবার মাঝে সমানভাবে ভাগ হয় (যেমন: খালার বিল, গ্যাস বিল) | ✅ হ্যাঁ, ফান্ড থেকে |

### ৩. ম্যানেজারের হাতে কত আছে (Manager Cash In Hand)

```
হাতে আছে = মোট জমা − (সাধারণ বাজার + অতিরিক্ত খরচ যেগুলো ফান্ড থেকে দেয়া হয়েছে)
```

**কোডে (DataContext.jsx, লাইন ৭৪-৭৭):**
```javascript
const messFundExpenses = expenses
    .filter(e => e.type !== 'baki' && e.fundSource !== 'own')
    .reduce((sum, e) => sum + Number(e.amount), 0);
const managerCashInHand = totalDeposit - messFundExpenses;
```

> **অর্থাৎ:** `baki` টাইপ বাদে এবং `own` (নিজের পকেট) বাদে, বাকি সব খরচ ফান্ড থেকে বাদ যায়।

### ৪. ফিক্সড বিল / ইউটিলিটি (Utilities)

Summary পেজে ফিক্সড বিল ক্যাটাগরি আছে (পানি, বিদ্যুৎ, ইন্টারনেট, ফ্রিজ ইত্যাদি)। এগুলো `utilities` ডকুমেন্টে সেভ হয় এবং সবার মাঝে সমানভাবে ভাগ হয়:

```
ইউটিলিটি প্রতি জন = মোট ইউটিলিটি ÷ মোট সদস্য সংখ্যা
```

### ৫. বাসা ভাড়া (House Rent)
- প্রতিটি সদস্যের জন্য আলাদা বাসা ভাড়া সেট করা যায়
- এটি সরাসরি সদস্যের `houseRent` ফিল্ডে সংরক্ষিত

### ৬. সদস্যের ব্যালেন্স ক্যালকুলেশন

```
ব্যালেন্স = (জমা + পকেট খরচ) − (খাবার খরচ + ইউটিলিটি শেয়ার + বাসা ভাড়া)

খাবার খরচ = সদস্যের মোট মিল × মিল রেট
মিল রেট = (মোট বাজার + বকেয়া) ÷ মোট মিল
```

## ❓ সমস্যা ১: খালার বিল / কারেন্ট বিল দেয়ার পর ফান্ড থেকে কিভাবে কাটবে?

### বর্তমান অবস্থা
এই ধরনের খরচ (যেমন: খালার বিল, কারেন্ট বিল) দুইভাবে হ্যান্ডেল হতে পারে:

#### উপায় ১: "অতিরিক্ত খরচ" হিসেবে (✅ ইতিমধ্যে আছে)
- **Expenses পেজে** → "কে বাজার করেছে?" ড্রপডাউন থেকে **"অতিরিক্ত খরচ (সবার মাঝে ভাগ হবে)"** সিলেক্ট করুন
- টাকার পরিমাণ ও বিবরণ দিন (যেমন: "খালার বিল মার্চ")
- এটি স্বয়ংক্রিয়ভাবে:
  - ✅ **ফান্ড থেকে কেটে যাবে** (managerCashInHand কমবে)
  - ✅ **সবার মাঝে সমানভাবে ভাগ হবে** (additionalExpensePerMember)
  - ✅ **প্রতিটি সদস্যের ব্যালেন্সে প্রভাব পড়বে**

#### উপায় ২: ফিক্সড বিল ক্যাটাগরি হিসেবে (⚠️ সরাসরি ফান্ড থেকে কাটে না)
- Summary পেজে ফিক্সড বিল (পানি, বিদ্যুৎ, ইন্টারনেট) এর ভ্যালু সেট করা হয়
- এগুলো **সবার মাঝে সমানভাবে ভাগ হয়** (utilityPerMember)
- কিন্তু এই বিলগুলো **সরাসরি ফান্ড থেকে কাটে না** — শুধু ব্যালেন্স ক্যালকুলেশনে ধরা হয়
- অর্থাৎ, ফিক্সড বিল সেট করলে ম্যানেজারের "হাতে আছে" অংক পরিবর্তন হয় না

### 💡 সুপারিশ

| কী ধরনের খরচ | কোথায় এন্ট্রি করবেন | ফান্ড থেকে কাটবে? |
|---|---|---|
| খালার বিল (একবার দিলেন) | Expenses → "অতিরিক্ত খরচ" | ✅ হ্যাঁ |
| কারেন্ট বিল (একবার দিলেন) | Expenses → "অতিরিক্ত খরচ" | ✅ হ্যাঁ |
| পানির বিল, ইন্টারনেট (প্রতি মাসে ফিক্স) | Summary → ফিক্সড বিল | ❌ না, শুধু ব্যালেন্সে |

> **সমস্যা:** ফিক্সড বিলগুলো (পানি, বিদ্যুৎ, ইন্টারনেট) যদি ম্যানেজার ফান্ড থেকে পে করেন, তাহলে সেগুলোকে **Expenses পেজে "অতিরিক্ত খরচ" হিসেবেও** এন্ট্রি করতে হবে — তাহলেই ফান্ড থেকে কাটবে।
>
> **অথবা** শুধু ফিক্সড বিলে ভ্যালু সেট করলেই হবে, কিন্তু তখন ম্যানেজারকে নিজে হিসাব করতে হবে যে ফান্ড থেকে কত বের হয়েছে।

### ✅ সেরা সমাধান (ভবিষ্যতে ইমপ্লিমেন্ট করতে পারি)

ফিক্সড বিলগুলোও অটোমেটিকালি ফান্ড থেকে কাটানোর অপশন যোগ করা:
- Summary পেজে ফিক্সড বিলের পাশে **"ফান্ড থেকে পে করা হয়েছে"** চেকবক্স
- চেক করলে ওই অ্যামাউন্ট `managerCashInHand` থেকে বাদ যাবে
- এতে দুইবার (ফিক্সড বিল + অতিরিক্ত খরচ) এন্ট্রি করার দরকার হবে না

## ❓ সমস্যা ২: কেউ বাসা ছেড়ে দিলে অগ্রিম বিল কিভাবে ভাগ হবে?

### পরিস্থিতি (উদাহরণ)

ধরি ৫ জন মেম্বার আছে। মাসের মাঝামাঝি **১ জন বাসা ছেড়ে দিচ্ছে**।

**চলে যাওয়া মেম্বারের ক্ষেত্রে:**
| হিসাব | দেবে? | কেন? |
|--------|-------|------|
| মিল খরচ (যতদিন খেয়েছে) | ✅ হ্যাঁ | সে মিল খেয়েছে, তাই তার মিল × মিল রেট ঠিকই কাউন্ট হবে |
| বাজার খরচ (পকেট থেকে করলে) | ✅ হ্যাঁ | তার নামে যে খরচ আছে সেটা ঠিকই থাকবে |
| অতিরিক্ত খরচ (খালার বিল ইত্যাদি) | ✅ হ্যাঁ | আগে যেগুলো ভাগ হয়ে গেছে সেগুলো সবাই দেবে |
| **কারেন্ট বিল (অগ্রিম)** | ❌ **না** | সামনের মাসে সে থাকবে না |
| **পানির বিল (অগ্রিম)** | ❌ **না** | সামনের মাসে সে থাকবে না |
| **ইন্টারনেট বিল (অগ্রিম)** | ❌ **না** | সামনের মাসে সে থাকবে না |
| **বাসা ভাড়া (অগ্রিম)** | ❌ **না** | সামনের মাসে সে থাকবে না |

**মূল সমস্যা:** এই মেম্বারের মিল খরচ ও অন্য সব খরচ ঠিকই আসবে, কিন্তু শুধু **অগ্রিম/ফিক্সড বিলগুলো** তার ওপর চার্জ হওয়া উচিত **না** — বাকি সদস্যদের মাঝে ভাগ হবে।

### বর্তমান সিস্টেমে কী হচ্ছে (সমস্যা)

```
বর্তমান ক্যালকুলেশন (DataContext.jsx):
────────────────────────────────────────
ইউটিলিটি প্রতি জন = মোট ইউটিলিটি ÷ সব মেম্বার সংখ্যা (members.length)
ব্যালেন্স = জমা − (মিল খরচ + ইউটিলিটি শেয়ার + বাসা ভাড়া)
```

- **ফিক্সড বিল:** `members.length` দিয়ে ভাগ হয় — **সবার ওপর সমানভাবে পড়ে**, চলে যাওয়া মেম্বারকে বাদ দেয়ার উপায় **নেই** ❌
- **বাসা ভাড়া:** প্রতি মেম্বারের আলাদা `houseRent` — ডিলিট করলেও বাকিদের ভাড়া **অটো বাড়বে না** ❌
- **মেম্বার ডিলিট করলে:** তার মিল, জমা, খরচ সব রেফারেন্স নষ্ট হয়ে যায় — হিসাব গড়বড় হবে ❌

### 💡 সমাধান: "বিল ছাড়" (Bill Exempt) সিস্টেম

**ধারণা:** মেম্বারকে ডিলিট না করে, শুধু ফিক্সড বিল থেকে বাদ (exempt) দেয়া। তার মিল খরচ, বাজার খরচ সব ঠিকই থাকবে।

#### কিভাবে কাজ করবে:

```
┌───────────────────────────────────────────────────────────────┐
│                    মেম্বার: রাফি                                │
├───────────────────────────────────────────────────────────────┤
│                                                               │
│  স্ট্যাটাস: ✅ Active                                         │
│  বিল ছাড়: 🔴 "ফিক্সড বিল থেকে বাদ"                          │
│                                                               │
│  ├── মিল খরচ:          ৩০ মিল × ৫০ = ১,৫০০ ✅ (ঠিকই আসবে)    │
│  ├── বাজার (পকেট):     ৫০০ ✅ (ঠিকই আসবে)                    │
│  ├── অতিরিক্ত খরচ:     ২০০ ✅ (আগের গুলো ঠিকই আসবে)          │
│  ├── কারেন্ট বিল:       ০ ❌ (বাদ)                             │
│  ├── পানির বিল:         ০ ❌ (বাদ)                             │
│  ├── ইন্টারনেট বিল:     ০ ❌ (বাদ)                             │
│  └── বাসা ভাড়া:         ০ ❌ (বাদ)                             │
│                                                               │
│  তার বিলের অংশ → বাকি ৪ জনের মাঝে ভাগ হবে                   │
│                                                               │
└───────────────────────────────────────────────────────────────┘
```

#### কোড পরিবর্তনের ধাপ:

**ধাপ ১: মেম্বারে `billExempt` ফিল্ড যোগ**
```javascript
// member object:
{ id: "abc", name: "রাফি", houseRent: 2000, billExempt: true }
```

**ধাপ ২: ক্যালকুলেশন পরিবর্তন (DataContext.jsx)**
```javascript
// বর্তমান:
const utilityPerMember = memberCount > 0 ? totalUtilityCost / memberCount : 0;

// নতুন:
const billableMembers = members.filter(m => !m.billExempt);
const billableMemberCount = billableMembers.length;
const utilityPerBillable = billableMemberCount > 0 ? totalUtilityCost / billableMemberCount : 0;
```

**ধাপ ৩: ব্যালেন্স ক্যালকুলেশন পরিবর্তন**
```javascript
const stats = members.map(m => {
    const mMeals = mealsByMember[m.id] || 0;
    const mCost = mMeals * mealRate;              // ✅ মিল খরচ ঠিকই আসবে
    const mHouseRent = m.billExempt ? 0 : Number(m.houseRent) || 0;  // ❌ exempt হলে ০
    const mUtility = m.billExempt ? 0 : utilityPerBillable;          // ❌ exempt হলে ০

    // বাকি হিসাব আগের মতোই...
    const balance = mTotalContribution - (mCost + mUtility + mHouseRent);
    // ...
});
```

**ধাপ ৪: বাসা ভাড়া রিডিস্ট্রিবিউশন**
```javascript
// exempt মেম্বারদের ভাড়া বাকিদের মাঝে ভাগ
const exemptRent = members
    .filter(m => m.billExempt)
    .reduce((sum, m) => sum + (Number(m.houseRent) || 0), 0);

const extraRentPerBillable = billableMemberCount > 0 ? exemptRent / billableMemberCount : 0;

// প্রতিটি billable মেম্বারের effective ভাড়া:
const effectiveRent = m.billExempt ? 0 : (Number(m.houseRent) || 0) + extraRentPerBillable;
```
